Lumen Notes

A local-first notes app for engineers who live in the terminal half the day and a browser the other half.

What it is

Lumen Notes is a note-taking application that stores every note as a plain Markdown file inside a directory you choose. There is no proprietary database, no required cloud account, and no lock-in: open the directory in any text editor and your data is still there, byte-for-byte. Sync, when enabled, runs through a CRDT layer over your own storage backend (S3-compatible, Dropbox, or a self-hosted server).

Key features

  • Local-first storage. Notes are Markdown files on disk. No vendor format.
  • Bi-directional links. [[wikilink]] syntax with automatic backlink panel and an interactive graph view.
  • Block-level transclusion. Embed a single paragraph or code block from one note into another and edits stay in sync.
  • Native Vim and Emacs keybindings. Modal editing is a first-class mode, not a plugin afterthought.
  • Plugin SDK in TypeScript. Third-party plugins run in a sandboxed worker; the SDK ships with type definitions and an integration-test harness.
  • End-to-end encrypted sync. Keys never leave the device. Sync is optional and works peer-to-peer or via a relay.
  • Offline-first. Every feature, including search and link suggestions, works without a network connection.

Pricing

Free for personal use. Pro tier ($8/month) adds the official sync relay, mobile clients, and priority support. A self-hosted license for teams is $200/user/year.

Drawbacks and known limitations

  • Steep learning curve. The keybinding-heavy interface is hostile to users who do not already know Vim or Emacs. Onboarding takes most users a full week.
  • No native mobile app on the free tier. Mobile access requires the Pro subscription, and even then the iOS app is read-mostly with limited editing affordances.
  • Plugin ecosystem is young. Roughly 80 plugins exist as of this writing, compared with thousands in more established competitors. Common workflows may require writing your own plugin.
  • Real-time collaboration is single-author per file. The CRDT layer handles merges across devices for the same user, but two humans cannot type into the same note simultaneously the way they can in a Google Doc.
  • Performance degrades past ~50,000 notes. Search remains usable but the graph view becomes sluggish, and startup time grows roughly linearly.
  • No web client. Desktop only (macOS, Linux, Windows). The team has stated a web client is "not on the roadmap".

Who it is for

Lumen Notes is aimed at individual technical users — software engineers, researchers, and writers — who value data ownership and keyboard-driven workflows over collaboration features. Teams that need shared real-time editing should look elsewhere.

Conclusion

If you already think in Markdown and want a notes app that will outlive any single company's funding round, Lumen Notes is a strong choice. If you primarily collaborate with others in real time, or you need a polished mobile experience out of the box, the trade-offs probably do not work in your favour.
