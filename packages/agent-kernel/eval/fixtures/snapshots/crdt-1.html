CRDTs: A Beginner's Primer

A Conflict-free Replicated Data Type, or CRDT, is a data structure designed to be replicated across multiple machines so that each replica can be updated independently and the replicas can be merged later without coordination, always converging on the same final state. CRDTs are the mathematical foundation of much modern collaborative software — multiplayer text editors, offline-first notes apps, and peer-to-peer databases.

The problem CRDTs solve

Suppose Alice and Bob both edit the same shopping list while offline. Alice adds "milk". Bob adds "eggs". When they come back online, both edits should appear in the shared list, and crucially, the order in which the merges happen should not change the final outcome. Naive approaches — last-writer-wins on the whole document, for example — would lose one of the edits. CRDTs provide an algebra in which the merge operation is commutative, associative, and idempotent, so the order and number of merges does not matter.

State-based vs operation-based

CRDTs come in two flavours. State-based CRDTs (CvRDTs) ship the entire current state of a replica when merging; the merge function is a join in a semi-lattice. Operation-based CRDTs (CmRDTs) ship individual operations, which must be commutative and delivered to every replica at least once. State-based variants are simpler to reason about; op-based variants are usually more bandwidth-efficient.

Common building blocks

  • G-Counter — a grow-only counter, each replica increments its own slot, merge takes the max per slot.
  • PN-Counter — two G-Counters (positive and negative); the value is positive minus negative.
  • G-Set — a grow-only set, merge is union.
  • 2P-Set — a set that supports removal by maintaining an "added" set and a "removed" set; once removed, an element cannot be re-added.
  • OR-Set — an observed-remove set, where each add is tagged with a unique identifier so an element can be re-added after removal.
  • LWW-Element-Set — last-writer-wins set, using timestamps to break ties.

Where to start reading

The original CRDT paper is "A comprehensive study of Convergent and Commutative Replicated Data Types" by Shapiro, Preguiça, Baquero, and Zawirski (2011). For a gentler introduction, the Hypermerge and Automerge documentation are both approachable and contain working code you can experiment with locally.
